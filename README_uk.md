# Мультифункціональний USB-міст для реверс-інжинірингу на RP2040

## Огляд

Пристрій підключається до ПК як **4 серійних порти** (USB CDC) та надає набір інструментів для дослідження невідомих плат та протоколів.

---

## USB CDC порти

| Порт | Назва | Призначення |
|------|-------|-------------|
| CDC0 | `SHELL` | Інтерактивний шелл — керування всім пристроєм |
| CDC1 | `UART0` | UART bridge |
| CDC2 | `UART1` | UART bridge |
| CDC3 | `PLOT`  | Текстовий вивід даних, сумісний з BetterSerialPlotter |

---

## Функціональні модулі

### UART Bridge (CDC1, CDC2)

- Два незалежних UART, **налаштування автоматичне** — пристрій зчитує line coding (baud rate, біти даних, парність, стоп-біти) який виставив комп'ютер на CDC порті і дзеркально застосовує до відповідного апаратного UART
- Прозора двостороння передача даних без участі користувача

### I2C

- **Scanner** — знаходить всі пристрої на шині, виводить адреси в SHELL
- **Sniffer** — пасивне прослуховування шини, вивід транзакцій в SHELL
- Shell команди: `i2c scan`, `i2c sniff start`, `i2c sniff stop`

## PLOT CDC — потоковий вивід (CDC3)

Формат рядка сумісний з **BetterSerialPlotter**:

```
pwm0:1250 pwm1:980 adc0:2.31 gpio0:1 gpio1:0
```

Кожен активний канал виводиться в рядок. Частота оновлення задається через SHELL.

### Керування каналами через SHELL

```
> plot pwm add 6        # додати PWM вимірювання на піні 6
> plot pwm remove 6
> plot adc add 26       # ADC потокове читання на піні 26
> plot gpio add 3       # GPIO моніторинг на піні 3 (до 2 пінів)
> plot rate 100         # частота оновлення в мс
> plot status           # показати активні канали
> plot clear            # зупинити всі канали
```

### PWM вимірювання

- Вимірює **частоту** та **duty cycle**
- Реалізація через вбудований PWM slice в режимі capture або PIO
- **Потокове читання** — дані йдуть в CDC3 `PLOT`

### ADC

- **Одиночне читання** через SHELL з виводом в SHELL: `adc read 26`
- **Потокове читання** — дані йдуть в CDC3 `PLOT`

### GPIO моніторинг

- Показує поточний стан (0/1)
- **Потокове читання** —  дані йдуть в CDC3 `PLOT`

## Архітектура прошивки

**FreeRTOS**, окремі таски:

| Таск | Відповідальність |
|------|-----------------|
| `usb_task` | TinyUSB polling |
| `shell_task` | Читання CDC0, парсинг команд |
| `uart_bridge_task ×2` | Моніторинг line coding, прозора передача CDC↔UART |
| `plot_task` | Збирає дані з модулів, формує рядок, пише в CDC3 |
| `i2c_task` | Scanner / sniffer, вивід в CDC0 |
| `measure_task` | PWM capture, ADC polling, GPIO polling |

## Підсумок пінів (приблизно)

| Функція | Піни |
|---------|------|
| UART0 | GP0, GP1 |
| UART1 | GP4, GP5 |
| I2C (SDA/SCL) | GP2, GP3 |
| PWM capture | GP6, GP7, GP12, GP13 |
| ADC | GP26, GP27, GP28 |
| GPIO monitor | GP14, GP15 |
